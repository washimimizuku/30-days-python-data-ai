# Day 10 Quiz: Mini Project

## Questions

### 1. Why is data validation important in a data pipeline?

**Your answer:**


**Correct answer:** Data validation ensures data quality by catching errors early, preventing bad data from corrupting analysis, and making the pipeline more robust. Invalid data can lead to incorrect insights and system failures.

---

### 2. What's the advantage of breaking code into functions?

**Your answer:**


**Correct answer:** Functions make code modular, reusable, testable, and easier to understand. Each function has a single responsibility, making debugging and maintenance simpler.

---

### 3. Why use JSON for exporting results?

**Your answer:**


**Correct answer:** JSON is human-readable, language-independent, widely supported, and perfect for structured data. It's ideal for APIs, configuration, and data exchange between systems.

---

### 4. How would you handle very large CSV files?

**Your answer:**


**Correct answer:** Process line-by-line instead of loading all into memory, use generators, consider chunking, or use specialized libraries like pandas for efficient processing of large datasets.

---

### 5. What improvements could you add to this project?

**Your answer:**


**Correct answer:** Add logging, command-line arguments, configuration files, more robust error handling, data visualization, automated testing, support for multiple file formats, or database storage.
